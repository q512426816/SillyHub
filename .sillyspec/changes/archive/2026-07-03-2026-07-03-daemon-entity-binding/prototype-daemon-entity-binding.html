③ 任务派发流程

workspace_member_runtimes.daemon_id + workspace.default_agent │ ▼ 在该 daemon_instance 的 daemon_runtimes 里 找 provider == default_agent 且 status == online │ ├─ 命中 → lease.runtime_id = 该 runtime（派发，记「哪个智能体跑的」） └─ 未命中 → 报错：「该守护进程未启用 ，请去守护进程启用 或本次指定其他智能体」

④ 注册与通信（守护进程 ↔ 后端）

• 配置隔离：守护进程配置按后端地址分文件 config-<server_hash>.json —— 连本地/连远程各一份，独立 daemon_local_id，解决同机多实例身份冲突。

• 注册上报：daemon_local_id + server_url + 探测到的 provider 列表 → 后端先 upsert daemon_instances，再为每个 provider upsert daemon_runtimes（带 daemon_instance_id）。

• WS 通道：从「每个智能体一条 WS」改为每个守护进程一条 WS（dict[daemon_id, WebSocket]），握手带 daemon_local_id。

• 心跳：守护进程单条心跳带 daemon_local_id + 各 provider 状态 → 后端更新 daemon_instances.last_heartbeat + 各 runtime.status。

• agent-detector 探测逻辑不动（仍扫 PATH），只改注册上报内容。

• breaking：守护进程与后端必须同步升级；数据倾向重置（CLAUDE.md 规则10 允许）。
