项目计划初始化从模板（原型）

变更 2026-07-17-project-plan-init-from-template · 方案 A（后端 service 自动）

1. 数据流：新建项目计划 → 按模板批量生成里程碑

计划节点模板（PlanNode，按 no 正序）
模板① 立项阶段 无模块
模板② 需求阶段 无模块
模板③ 设计阶段 无模块
模板④ 实施阶段 有模块
↓ create_ps_project_plan 同事务批量建
项目计划 PsProjectPlan + 里程碑 PsPlanNode × 4
里程碑① 立项 has_module=false template_plan_node_id=模板①
明细：复制模板① 的 PlanNodeDetail × N（status=draft, module_id=null）
里程碑② 需求 false + 明细草稿
明细：复制模板② 的明细（draft）
里程碑③ 设计 false + 明细草稿
明细：复制模板③ 的明细（draft）
里程碑④ 实施 has_module=true template_plan_node_id=模板④
（空，不建明细——等新建模块时复制）
关键：has_module=无 → 里程碑 + 明细（草稿）；has_module=有 → 只建空里程碑。
所有自动创建的里程碑明细 status=draft（草稿）。

2. 实施阶段里程碑：新建模块 → 复制模板明细

里程碑④ 实施 has_module=true（空，来自模板④）
↓ 用户点"新建模块"（create_module 检测 template_plan_node_id → 模板④）
新模块 PlanNodeModule（挂里程碑④） module_name=前端...
↓ 同事务复制模板④ 的全部 PlanNodeDetail
明细：复制模板④ 明细 → PsPlanNodeDetail（status=draft, module_id=新模块.id）
手动建的里程碑（template_plan_node_id=null）→ 新建模块为空，不复制。

模板 PlanNode→ 里程碑 PsPlanNode

4. 前端 milestone-details 模块层

展示条件：overall_stage === "实施阶段" → 改为 PsPlanNode.has_module === true

  • has_module=true 的里程碑 → 三级（里程碑 → 模块 → 明细），有"新建模块"入口
  • has_module=false 的里程碑 → 二级（里程碑 → 明细）

项目计划创建流程零改动（后端自动建里程碑）。

5. 零回归

  • 现有项目计划/里程碑不动（只对新建项目计划生效）
  • 模板表 PlanNode/PlanNodeDetail/PlanNodeModule 不动
  • importer、ps 簇其他逻辑不动
  • 手动建的里程碑（template_plan_node_id=null）行为不变
