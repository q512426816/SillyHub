整体流程图

新成员加入工作区 │ ▼ [未初始化] ──点「初始化」──▶ 派 init 任务 ──▶ daemon 写 .sillyspec-platform.json │ + pull 文档缓存 + 回灌本地改动 ▼ [已初始化·未扫描] ──提示「请先扫描」──▶ 任一成员点「扫描」──▶ 生成文档回灌服务器 │ │ │ ▼ ▼ 文档落服务器（权威） [已扫描·就绪] ◀──────────────────────────── 拉文档 ◀────┘ │ ▼ 日常：每次任务前查版本保鲜 ｜ 服务器权威 / 客户端缓存 ｜ 整包同步（先够用再说）

关键决策（D-001 ~ D-010）

  • D-001 文档同步：整包（YAGNI，留 spec_version 扩展点）
  • D-002 初始化按钮：重定义为「下发 .sillyspec-platform.json + 拉文档缓存」
  • D-003 扫描归属：任意成员可扫 + 已有则提示
  • D-004 扫描有效期：永不过期（有文档即已扫）
  • D-005 初始化 vs 扫描：初始化只拉已有，无则提示先扫；扫描独立
  • D-006 / D-007 per-member 接线（复用 WorkspaceMemberRuntime）+ 客户端路径 per-member 可编辑
  • D-008 双向冲突：pull 前先 push 本地改动（服务器权威，本地不丢）
  • D-009 / D-010 配置下发走 init 任务（lease）+ 缓存按 spec_version 检查保鲜
