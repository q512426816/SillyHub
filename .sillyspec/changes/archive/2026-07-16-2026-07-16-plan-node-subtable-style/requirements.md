# 需求（Requirements）— 计划节点模板子表样式优化

> 变更 `2026-07-16-plan-node-subtable-style` · scale: small

## FR-001 子表独立横向滚动

展开任意模板行后，明细子表、模块子表**各自有独立的横向滚动条**；拖动子表滚动条只滚动该子表，不影响母表；母表横向滚动条恢复为母表列宽决定的正常长度（不再被子表撑长）。

## FR-002 明细列宽压缩

明细子表 7 列列宽整体紧凑：详细阶段 90 / 任务主题 100 / 任务描述 140 / 要求与注意事项 120 / 角色 80 / 成果 90 / 总体阶段 90（操作列 PpmSubTable 自加，不变），合计 ≤ 790px（现状 1000px）。任务描述仍为最宽列，保留 textarea 多行展开。

## FR-003 改动范围限定

仅修改 `frontend/src/app/(dashboard)/ppm/plan-nodes/page.tsx`；**禁止**改动 `components/ppm-sub-table.tsx` 通用组件、母表列结构、后端接口与数据模型。`milestone-details`、`ppm-project-plan-detail` 行为不变（零回归）。

## FR-004 视觉一致

明细、模块两个子表均套滚动容器，展开区内视觉风格统一（容器边界、滚动行为一致）。

## 验收标准

- 展开模板行：母表无超长横向滚动条；明细/模块子表底部各有独立横向滚动条；
- 列宽紧凑，任务描述可多行编辑；
- 其他 ppm 页面无视觉/行为变化。
