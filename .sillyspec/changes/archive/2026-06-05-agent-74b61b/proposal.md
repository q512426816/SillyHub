# Proposal: Agent 控制台日志回显宽度调整

## 动机

Agent 控制台中"已完成运行"的日志查看功能，日志回显区域宽度被 `max-w-6xl`（1152px）限制，导致长日志行在容器内显示时需要频繁水平滚动，影响日志阅读效率。

## 关键问题

1. **长日志行被截断**：Agent 运行日志中常见长文本行（如 TOOL_USE 命令、API 请求路径、JSON 参数），在 1152px 限制下这些行被截断或需要横向滚动
2. **空间浪费**：AppShell 主内容区在 1920px 屏幕上有约 1660px 可用空间，但日志区域仅使用约 1104px（1152px - padding），浪费了约 34% 的可用宽度
3. **调试效率低**：Agent 控制台是开发调试的核心工具，日志可读性直接影响排查问题的速度

## 变更范围

- 移除 `frontend/src/app/(dashboard)/workspaces/[id]/agent/page.tsx` 页面根容器的 `max-w-6xl` 宽度限制
- 同时移除无意义的 `mx-auto` 居中对齐

## 不在范围内（显式清单）

- 不修改日志渲染逻辑（`renderConversationLog` 函数）
- 不修改日志高度限制（`max-h-[300px]`）
- 不修改活跃运行（Active Runs）区域的 UI
- 不修改后端 API 或数据模型
- 不添加全屏模式切换按钮
- 不修改其他页面的宽度设置
- 不修改 AppShell 布局或 sidebar 宽度

## 成功标准（可验证）

1. 在 1920px 屏幕上，日志展示区域宽度接近 viewport 减去 sidebar 的宽度（约 1660px），不再被 1152px 截断
2. 长日志行（>200 字符）在容器内可完整显示或自然折行，无需水平滚动
3. 页面其他元素（头部、表格、按钮）功能不受影响
4. 在 1280px 屏幕上，页面布局正常，内容不溢出
