设计要点回顾（展开）
• ALLOW+DENY 全量记录（D-006），backend 落 PolicyAuditLog 表，daemon 批量上报 + 限流
• 按 runtime 隔离（D-002）：审计页进入即绑定某 runtime，只看该 runtime 的记录
• 「Agent 种类」= provider（claude/codex/...），项目无独立 Agent 实体
• 路径规范化含 realpath（D-005），展示规范化后的路径
• Python/Node 脚本内部 open() 不出现在此页（D-001 降级，靠 prompt + 无法 audit 孙进程）
