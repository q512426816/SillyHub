# 提案书（Proposal）

## 动机

问题清单（`/ppm/problem-list`）目前只能逐条「新建问题」录入。在批量录入场景（如某项目一次性梳理几十条历史问题、跨项目集中录入）下，逐条填表单效率极低，成为使用瓶颈。本次新增 Excel 批量导入，解决「批量录入麻烦」的核心痛点。

## 关键问题（现有方案为何不够）

1. **无批量入口**：问题清单只有「新建问题」表单（`ProblemDrawer`），一条一条填，字段多（17 个业务字段）时重复劳动大。
2. **跨项目录入需反复切换**：问题清单页是跨项目的（项目是筛选项），逐条新建时每条都要在表单里选项目，无法一次性录入多个项目的问题。
3. **缺少入库前校验预览**：批量录入若直接落库，姓名/项目填错会产生脏数据且难发现；需要「先预览、未匹配标红、确认后再入库」的安全机制。

## 变更范围

为问题清单新增 Excel 批量导入功能，对齐已有「项目计划导入」的两步式范式：

- **后端**：`ppm/common/upload.py` 通用上传校验 + `problem/importer.py` 纯解析（枚举规范化）+ `import-preview`/`import-commit` 两个端点 + service（批量反查 项目/模块/项目成员 + 严格校验 + 原子单次事务入库 + date→datetime 转换 + commit 重查防篡改）。
- **前端**：三态导入弹窗（上传 → 预览标红 → 结果统计，复制项目计划导入弹窗范式）+ 问题清单页接入「导入」按钮 + 静态模板下载。
- **校验**：项目名/责任人/验证人/模块名填了但匹配不到系统记录 → 整行标红拒绝；必填 = 项目名 + 问题描述；不查重。

## 不在范围内（显式清单）

- **不做**附件导入（`file_urls`，导入后可在页面补）。
- **不做**查重（Excel 几行建几条）。
- **不做**导入「进行中/已完成」态问题（只产「新建」态）。
- **不做**动态下拉模板（静态模板 + 严格校验已够）。
- **不改**现有 `create_problem` / 导出 / 列表查询 / 3 态执行流 API 与表结构。

## 成功标准（可验证）

- 旧功能（新建/编辑/导出/列表/执行流）行为完全不变。
- 新增「导入」按钮可用，可下载 17 列全字段模板。
- 上传 Excel 后预览：项目名/责任人/验证人/模块名填了未匹配的行、项目名或问题描述为空的行 → 标红且不可导入。
- 确认导入后，可导入行**原子入库**（全成或全回滚），新建态、创建人=当前用户。
- 支持一次导入跨多个项目的问题（Excel 每行项目名反查）。
- 前端无法通过篡改回传 UUID 越权（commit 重新反查 + data_scope 校验）。
- 后端 `app/modules/ppm` 测试 + 前端组件测试通过。
