2026-06-18-workspace-client-path — 设计原型

Workspace 支持「daemon 客户端路径」| 线框原型，用于确认设计方向，非最终 UI

① 架构总览

  浏览器 (Next.js)                Backend (FastAPI)                 Daemon 客户端 (Node CLI)
  ─────────────────              ──────────────────                ───────────────────────
  创建 workspace  ──POST /workspaces──▶  Workspace 表
   ├ 路径来源                          + path_source                (代码 + .sillyspec 在
   │  = server-local                   + daemon_runtime_id ◀─强绑────  这台机器，backend 读不到)
   │  = daemon-client                       │
   │                                        ├─ list_dir ──WS RPC──▶ readdir+stat
   │                                        │   POST /runtimes/{id}    ▕ allowed_roots 白名单校验
   │                                        │   /list-dir              ▕ 越界 → 403
   │                                        │                          │
   │                                        │   agent run 路由          │
   │                                        ├─ dispatch_to_daemon ───▶ 按 workspace.daemon_runtime_id
   │                                        │   (覆盖 user 级选择)      │ 定向，离线 → fail
   │                                        │                          │
   │                                        │   spec 按需下发(方案A)     │
   │                                        ├─ GET /spec-ws/{id}/bundle▶ 解到 ~/.sillyhub/daemon/specs/{id}
   │                                        │                          │ 起 agent (spec_root=本地临时区)
   │                                        │   ◀──POST /spec-ws/{id}/sync── 执行后整树回传

真理源：服务器 spec_root（A1）。daemon 仅「执行时借阅 spec 到本地临时区」。

② 数据模型变更（Workspace 表）

字段类型含义约束
root_pathText代码根目录绝对路径现有；server-local=backend 可访，daemon-client=daemon 机器路径
path_source ✚str(20)路径来源server-local / daemon-client，默认 server-local
daemon_runtime_id ✚UUID?绑定 daemon (D-001)path_source=daemon-client 时必填
✚ = 新增列。现有 server-local workspace 默认 path_source=server-local、daemon_runtime_id=NULL，行为零变化。

④ 关键流程

list_dir 目录浏览（前端交互）

→
② POST /api/daemon/runtimes/{id}/list-dir {path}
→
③ backend 经 WS 发 RPC 给该 daemon
→
④ daemon readdir+stat，校验 allowed_roots
→
⑤ WS RPC_RESULT 回传 → 前端渲染子节点
agent run + spec 按需下发（方案A）
① 发起 agent run
→
② 按 workspace.daemon_runtime_id 路由（离线则 fail）
→
③ daemon GET /spec-ws/{id}/bundle → 解到本地临时 spec_root
→
④ 起 agent（cwd=root_path, spec_root=本地临时区）
→
⑤ POST /spec-ws/{id}/sync 回传 → 服务器覆盖 + 重parse
