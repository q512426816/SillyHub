# task-04: 浏览器视觉验证三处日志区域

## 修改文件（必填）
- 无（纯验证任务）

## 实现要求

### 前置条件

1. 确认 task-01、task-02、task-03 已全部完成（读取对应的 task-N.md 确认状态，或检查代码变更是否已落地）
2. 确认前端开发服务器可正常启动：`cd /host-projects/WorkNew/SillyHub/frontend && pnpm dev`（默认端口 3000）
3. 准备测试数据：至少需要一个包含 Agent 运行日志的 workspace，且有超宽日志内容（如长路径、工具调用参数等）。如果没有现成数据，可通过以下方式制造：
   - 运行一个会产生长输出日志的 Agent 任务
   - 或通过浏览器 DevTools 在日志 DOM 元素中临时注入超长文本来模拟

### 验证页面与步骤

#### 页面 A：Agent 控制台 — 活跃运行日志

- **URL**：`/workspaces/[id]/agent`（需要有活跃运行的 Agent）
- **验证内容**：
  1. 滚动到"活跃运行日志"区域（section，外层卡片容器）
  2. 检查日志内容行：超长日志文本是否在 span 内产生水平滚动条（对应 task-01 修复的 `overflow-x-auto`）
  3. 检查页面整体：是否存在 X 轴滚动条
  4. 检查 channel 着色：INFO（默认色）、TOOL（蓝色）、WARN（黄色）、INPUT（绿色）、SENT（紫色）等标签着色是否正常
  5. 检查自动滚动：新日志追加时，是否自动滚动到底部

#### 页面 B：Agent 控制台 — 已完成运行日志

- **URL**：`/workspaces/[id]/agent`（点击已完成运行列表中某条记录的"查看日志"按钮）
- **验证内容**：
  1. 展开已完成运行的日志面板（点击"查看日志"按钮后显示的 `<pre>` 区域）
  2. 检查超长日志文本是否在 `<pre>` 块内产生水平滚动条（对应 task-02 修复）
  3. 检查页面整体：是否存在 X 轴滚动条
  4. 检查日志内容完整性：超长文本无截断，水平滚动可查看全部内容
  5. 检查 channel 着色是否正常

#### 页面 C：变更详情页日志查看器

- **URL**：`/workspaces/[id]/changes/[cid]`（需要有包含 Agent 运行日志的变更）
- **验证内容**：
  1. 找到"日志查看器"区域
  2. 检查超长日志文本是否在日志内容区域产生水平滚动条（对应 task-03 修复的 `overflow-x-auto`）
  3. 检查页面整体：是否存在 X 轴滚动条
  4. 检查日志内容完整性：超长文本无截断
  5. 检查 channel 着色和标签显示是否正常

### 多分辨率验证

在以下两个分辨率下分别执行上述验证：
- **1280px 宽度**（模拟笔记本电脑）：更容易触发日志溢出
- **1920px 宽度**（模拟桌面显示器）：确认大屏下表现正常

可通过 `mcp__plugin_playwright_playwright__browser_resize` 调整浏览器窗口大小。

## 接口定义

纯验证任务，无代码接口。

## 边界处理

1. **前置任务未完成**：在开始验证前，检查 task-01/02/03 的代码变更是否已实际落地。如果发现任何前置任务未完成，记录缺失项并停止验证，反馈给对应任务
2. **前端服务器启动失败**：检查端口 3000 是否被占用，尝试 `lsof -i :3000` 查看占用进程，必要时 kill 后重新启动
3. **无活跃运行数据**：如果没有活跃的 Agent 运行，可改用已完成运行的日志面板验证（页面 B），或通过浏览器 DevTools 注入超长文本来模拟。具体方法：在浏览器中选中日志内容 span 元素，通过 `mcp__plugin_playwright_playwright__browser_evaluate` 注入 `element.textContent = 'A'.repeat(500)` 模拟超长内容
4. **多分辨率验证**：先在 1280px 下完成全部验证，再切换到 1920px 重复关键验证点。确保两档分辨率下行为一致
5. **验证失败处理**：记录失败的具体页面、具体验证点、失败现象描述（如"页面出现 X 轴滚动条"或"日志内容被截断"），并截图保存。反馈给对应 task（task-01/02/03）进行修复
6. **浏览器 DevTools 模拟超长文本**：使用 `browser_evaluate` 在日志内容 span 上设置超长文本，确保验证不依赖特定数据状态
