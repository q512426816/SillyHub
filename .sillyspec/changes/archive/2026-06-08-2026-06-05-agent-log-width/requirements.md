# Requirements: Agent 控制台日志回显宽度修复

## 角色

| 角色 | 说明 |
|---|---|
| 开发者 | 在 Agent 控制台查看 Agent 运行日志的用户 |

## 功能需求

### FR-01: 日志块内水平滚动

**Given** Agent 控制台或变更详情页有日志显示区域
**When** 日志内容行的宽度超过容器宽度（如长路径、工具调用参数）
**Then** 日志块内出现水平滚动条，用户可在日志块内左右滚动查看完整内容

### FR-02: 页面无 X 轴滚动条

**Given** 用户在 Agent 控制台或变更详情页查看日志
**When** 日志内容行超宽
**Then** 页面本身不出现水平滚动条

### FR-03: 日志内容完整性

**Given** 日志内容包含超长文本行
**When** 用户在日志块内水平滚动
**Then** 所有内容完整显示，无字符截断或丢失

### FR-04: 现有功能不受影响

**Given** 日志显示区域的 channel 着色、标签、自动滚动等功能
**When** 添加 overflow 修复后
**Then** 着色、标签、自动滚动等行为与修复前一致

## 非功能需求

- **兼容性**：不修改任何 API、数据结构或组件接口
- **可回退**：纯 CSS 类名变更，可立即回退
- **可测试**：通过浏览器视觉验证，无需额外测试工具
- **性能**：无性能影响
