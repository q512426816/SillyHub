Codex Interactive Session Flow

目标：让 /runtimes 的 Codex 会话复用 AgentSession 生命周期，达到 Claude Code 同等级的多轮、打断、结束、历史与恢复能力。

端到端链路

FrontendRuntimeSessionDialog / InteractiveSessionPanel
BackendAgentSession + AgentRun + interactive lease
DaemonSessionManager provider driver
Codex Driverapp-server stdio JSON-RPC v2
Codex CLIthread + turn runtime
create / inject / interrupt / end / reopen / recover 走同一条 session 控制链

Provider Driver 分层

Claude
ClaudeSdkDriver 保持现有 SDK query / interrupt / resume 行为。
现有消息映射、AskUserQuestion、partial streaming 不改。
Codex
CodexAppServerDriver 长驻 app-server 进程。
thread/start 建 thread，turn/start 发首轮和后续轮。
turn/interrupt 打断当前 turn，thread/resume 恢复历史 thread。

会话状态

pending后端已建 session/run/lease
active可接收下一轮 inject
running当前 turn 执行中，可 interrupt
reconnectingdaemon 恢复 / reopen 中
ended/failed终态，可历史回看；支持 reopen

关键验收

  • Codex runtime 会话按钮进入 interactive session 面板，不再走 quick-chat。
  • 同一 Codex AgentSession 可连续 inject 多轮，历史按 run_id 分 turn 展示。
  • 运行中可点击打断，当前 turn 收敛后 session 仍可继续。
  • 结束 session 后可回看历史；reopen 使用 Codex threadId 恢复。
  • Claude Code 现有 create/inject/interrupt/end/reopen/recovery 测试保持通过。
