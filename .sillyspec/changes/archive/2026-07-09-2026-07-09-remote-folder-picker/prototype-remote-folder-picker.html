RemoteFolderPicker · 原型

变更 2026-07-09-remote-folder-picker · 基于 Daemon 的远程目录浏览器（替换 browse_folder 系统弹窗）

📐 数据流 list_roots 新增 list_dir 复用 PUT allowed-roots → policy_update 复用

RemoteFolderPicker(runtimeId) ├─ 打开 listRoots(rid) → POST /list-roots → daemon list_roots │ Win → { roots: ["C:\\","D:\\","E:\\"] } │ Unix→ { roots: ["/"] } ├─ 展开 listDir(rid, path) → POST /list-dir → daemon list_dir (已有，返 entries[{name,type}]，只取 dir) ├─ 手输 跳转前探 listDir 校验 → not_found 则提示「路径不存在或非目录」 └─ 确认 onChange(path) → 调用方 PUT /allowed-roots → WS policy_update → daemon 即时生效 ❌ 已移除：browse_folder（daemon 弹 PowerShell Shell.BrowseForFolder，远程 daemon 时 Web 用户看不到）
新增 复用已有 移除

🖥️ Runtime 配置页（接入点）

每个可写目录行旁的「浏览」按钮打开 RemoteFolderPicker。组件自治，调用方只传 runtimeId + 受控 value/onChange。

模拟 daemon 平台（决定 list_roots 返回）：

保存 → PUT /runtimes/{id}/allowed-roots → WS policy_update（在线秒级生效；离线下次心跳兜底）。

🧩 RemoteFolderPicker 组件 · 关键状态

懒加载展开（antd Tree loadData） 地址栏手输 + 跳转校验 已选路径回显 错误降级不崩溃

点击上方任意「浏览」按钮体验完整交互。切换平台看 list_roots 根锚点差异；勾选「daemon 离线」看错误降级。
