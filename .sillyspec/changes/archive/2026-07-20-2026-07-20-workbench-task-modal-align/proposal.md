# proposal.md — 工作台「我的任务」操作弹窗对齐任务计划页

> 变更 `2026-07-20-workbench-task-modal-align` · 纯前端重构（antd Modal 抽公共组件复用）

## 1. 背景

个人工作台 `/ppm/workbench` 的「我的任务」表格，其「详情 / 执行 / 启动」操作按钮触发的弹窗，与任务计划页 `/ppm/task-plans` 对应操作的行为严重不一致：

- **任务计划页**（2026-07 后升级）：点击「详情/执行」打开一个带任务完整上下文的 antd `Modal`（任务信息卡 + 历史执行记录表 + 跨天拆分填报区），「启动」只切换状态不弹窗。逻辑内联在 `task-plans/page.tsx` L263-392、L792-969。
- **工作台**（停留在旧方案）：「详情」是个只读字段 `Dialog`（无执行记录表），「执行/启动」走简陋的 `ExecuteTaskDialog`（只填一次耗时、看不到历史记录、不支持跨天拆分）。

两处本应一致的「任务执行」体验出现分叉。根因：任务计划页升级时未把详情 Modal 抽成共享组件，工作台仍用旧 `ExecuteTaskDialog`。

## 2. 目标

- 抽公共组件 `TaskDetailModal`，任务计划页与个人工作台两边复用，单一真实源，彻底消除分叉
- 工作台「详情/执行」弹出与任务计划页**逐字一致**的任务详情窗（任务信息卡 + 历史执行记录表 + 跨天拆分填报）
- 工作台「启动」对齐任务计划页：仅 `startPlanTask` 切状态 + toast，不弹执行填写窗
- 清理遗留：删除任务计划页未被调用的 `handleResume` + `ExecuteTaskDialog` 引用；若 `ExecuteTaskDialog` 抽取后成孤儿则一并删除

## 3. 非目标

- ❌ 不改后端 / API / 数据流 / 数据库
- ❌ 不改任务计划页现有行为（抽组件后必须与现状逐字一致，verify 防回归）
- ❌ 不给工作台引入「编辑 / 删除」任务能力（那是管理者视角，个人工作台只保留 详情/启动/执行）
- ❌ 不改 `ExecuteTaskDialog` 组件内部逻辑（仅判断去留）
- ❌ 不引入新路由 / 新状态管理

## 4. 方案选择

| 方案 | 说明 | 取舍 |
|---|---|---|
| **A（选定）** | 抽公共 `TaskDetailModal`，两页复用 | 单一真实源、两页天然一致；代价是动任务计划页有回归风险，靠 verify 全量测试兜底 |
| B（否决） | 工作台内联复制任务计划页 ~200 行 Modal 逻辑 | 不动任务计划页零回归，但重复代码未来必漂移（正是当前问题根源） |
| C（否决） | 工作台点详情/执行跳转任务计划页 | 零代码重复，但离开工作台体验差，用户 AskUserQuestion 已否决 |

用户已通过 AskUserQuestion 确认选 A。

## 5. 验收口径

- 工作台「我的任务」点「详情」→ 弹出与任务计划页同款任务详情窗（任务信息卡 + 执行记录表），只读
- 工作台「我的任务」点「执行」（进行中任务）→ 弹出同款窗 + 填报区（跨天拆分，提交回未开始 / 完成两种动作）
- 工作台「我的任务」点「启动」→ 仅 toast 提示 + 列表状态刷为进行中，不弹窗
- 任务计划页详情/执行/启动行为与改造前**完全一致**（前端全量单测零回归 + 浏览器实测）
- `tsc --noEmit` 通过
