# Proposal: 用户管理模块升级 v2（增量）

## 变更名
`2026-06-10-user-management-v2`

## 动机

第一阶段补齐了权限校验和安全保护，但会话管理仍不完整：无法单独撤销某个会话，无法在 UI 查看用户所属 Workspace，操作列管理操作暴露在列表中不够简洁。

## 关键问题

1. 管理员无法精准撤销单个可疑会话（只能通过禁用用户间接撤销全部）
2. 前端操作列过于密集，管理操作（禁用/删除）和查看操作混杂
3. 无法查看用户属于哪些 Workspace 及角色

## 变更范围

- 单个会话撤销端点 + 批量撤销端点
- 密码重置审计标记（force_change_on_next_login）
- 用户 Workspace 角色查询端点
- 前端操作列简化 + Drawer 增强

## 不在范围内

- 邀请用户流程
- Git 身份绑定
- 自定义角色编辑
- MFA
- force_change_on_next_login 持久化（仅审计标记）

## 成功标准

- 管理员可撤销单个或全部会话
- 用户详情 Drawer 显示所属 Workspace
- 操作列只有"详情"链接
- 所有新增操作写入审计日志
- 现有 API 兼容
