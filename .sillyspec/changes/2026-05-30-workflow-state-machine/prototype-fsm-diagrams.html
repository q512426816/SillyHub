4. Review + Reject Flow

  Reviewer submits review (approve / reject)
       │
       ├── verdict = "approve"
       │     └── Review record saved, no auto-transition
       │
       └── verdict = "reject"
             ├── If FSM allows → rejected state (auto-transition)
             └── Review record saved with comment (reason)
                    │
                    └── Change can be reworked:
                        rejected → draft → proposed → ... (restart)
