# 提案书（Proposal）— 里程碑明细·实施阶段 模块导入

## 动机
实施阶段里程碑下的「明细·模块」两级数据（模块 `PlanNodeModule` → 明细 `PsPlanNodeDetail`）目前只能逐条手工新建。项目从外部《项目详细开发计划》Excel 迁移时，几十上百条模块/明细要逐条录入，效率低、易错、难与原始计划保持一致。业务方已有标准 Excel 模板（区分「正常计划」与「临时插单」），需要一个批量导入入口。

## 关键问题（现有方案为何不够）
1. **逐条录入效率低**：一个实施阶段里程碑常含数十条模块+明细，手工新建每条都要填模块名/明细阶段/主题/描述/工作量/责任人/起止日期，单次迁移耗时且易漏填错填。
2. **无法区分计划类型**：现有模块表无「计划类型」字段，正常计划与临时插单混在一起无法区分，影响排期与统计。
3. **Excel 数据无法批量映射**：Excel 的「平台/子系统→模块、任务分类→明细阶段、任务主题/描述/工作量/责任人/起止」没有自动落库通道，全靠人工搬。

## 变更范围
- **后端**：`PlanNodeModule` 新增 `plan_type` 字段 + alembic 迁移；新建 Excel 解析模块 `importer.py`（按表头名匹配列）；service 加 `import_preview`/`import_commit`（责任人反查、同名合并、模块汇总、单事务原子提交）；router 加两个端点；新增 `python-multipart` 依赖。
- **前端**：`ModuleLevelTable` 加「导入模块」按钮 + 「计划类型」列；导入弹窗（上传→预览勾选 Sheet+标错→确认→结果报告）；FormData 上传函数。
- **测试**：解析/反查/入库单测 + 端点集成测试。

## 不在范围内（显式清单）
- 不导入：周次、状态、计划执行情况、评估说明、备注。
- 不支持 `.xls`（老格式）与 CSV（本期不做）。
- 不改「模板簇」（`plan-nodes` 页）的模块表，只覆盖 ps 执行簇的「实施阶段里程碑」。
- 不做导入撤销/回滚按钮（依赖事务原子性；误导入手动删）。
- 不做异步导入队列/进度条（单文件量级小，同步线程池即可）。
- `ModuleFormDrawer` 是否暴露「计划类型」编辑器为可选增强，留 plan 阶段定夺。
