③ 会话状态机

create first turn result ┌─────────────┐ ──────► ┌──────────┐ ◄──────────────────┐ │ pending │ │ active │ │ inject(下一轮) └─────────────┘ └────┬─────┘ │ │ │ ┌────────────────┼───────────────┐ │ interrupt │ │ end │ failed │ (本轮SIGINT)│ │ (kill进程) │ (崩溃) │ ▼ ▼ ▼ │ (仍 active, ┌──────────┐ ┌──────────┐ │ 可继续追问) │ ended │ │ failed │ │ └──────────┘ └──────────┘ │ ▲ │ │ Wave3: resume 持久化后可重连 │ │ daemon 重启 --resume 重建 ───┘
打断 vs 结束分离：打断本轮 = SIGINT/turn interrupt（保留会话）；结束会话 = kill 进程（标 ended）

④ 实施路线（Wave 分组，execute 分波次推进）

Wave 1 · 核心交互
agent_sessions 表 + lease.kind + task-runner session 模式(result 不 end) + WS session_inject/interrupt/end + sessionStore + 进度复用 SSE
→ 中途追问/多轮注入(核心卖点)
Wave 2 · 权限往返
manual_approval 开关 + WS permission_request/response + stream-json control_request 升级为暂停等远程
→ 手动批准模式
Wave 3 · resume
daemon 磁盘持久化 session 状态 + 重启 --resume 重 spawn(claude)/thread resume(codex)
→ 崩溃恢复
Wave 4 · 前端管控台
runtimes 页 quick-chat 升级会话面板 + 历史回看 + 权限弹窗
→ 可视化交互
本原型为线框预览，重点确认设计方向（架构数据流 / 会话面板 / 状态机 / Wave 分层），非最终 UI。核心改造在 daemon+backend 协议层。
