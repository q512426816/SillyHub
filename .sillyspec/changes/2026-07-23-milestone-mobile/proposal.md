# 提案书（Proposal）— 里程碑明细移动端整页完整复刻

## 动机
PPM「里程碑明细」是项目计划里功能最全、最核心的管理页（桌面 `milestone-details/page.tsx`，2988 行：三层嵌套 + 8 mode 表单 + 审批流程 + Timeline + 工作日 + 版本链 + Excel 导入导出）。移动 APP 架构已落地（`/m/` middleware + components/mobile 通用件 + 底部 5 Tab），并已上线项目计划、问题清单、任务计划、工作台四个移动页，但最复杂的里程碑明细还没上移动。现场的项目执行人和审核人（工地、出差）只能用手机，却无法在手机上开立、变更、审批明细，移动办公闭环缺最后一块。

## 关键问题
1. **现场移动办公断档**：项目执行人/审核人在现场只能用手机，但里程碑明细的开立、变更、审批全在电脑版，被迫回电脑或直接延误。
2. **电脑页在手机上不可用**：桌面版是行内展开 + 横向表格 + 多块复杂表单，手机竖屏打开后排版崩、点不动，等于没有移动能力。
3. **现有移动页覆盖不全**：已有计划/问题/任务/工作台四个移动页，缺最复杂的里程碑明细，移动办公闭环不完整。

## 变更范围
新增移动里程碑明细整页（`app/m/ppm/milestone-details/page.tsx` + 7 个 mobile 组件），实现：
- 三层钻取（里程碑 → 模块 → 明细），竖屏一层层点进去
- 8 mode 表单（create/edit/changeInfo/audit/approve/change/changeApprove/view），按状态分发
- 里程碑 / 模块 / 明细 各自 CRUD
- 审批流程（save/reject/change），表单内提交
- Timeline 履历 + 工作日自动算 + 版本链
- 模块 Excel 导入 + 明细导出
- 权限（readOnly 总开关 + 块级 mode/user 匹配）
- 入口接到项目计划移动卡片

纯前端，100% 复用 `lib/ppm/plan.ts` 现有接口和 `components/mobile` 通用件，不写新后端，不动桌面。

## 不在范围内（显式清单）
- 不改桌面 `milestone-details/**`（零回归）
- 不写新后端 API / 数据模型 / migration
- 不做横屏表格（只竖屏卡片式钻取）
- audit / approve / changeApprove 三个 mode 当前线上很少触达，本变更只预留入口与只读展示，不做完整可交互 UI（见 design D-005）

## 成功标准（可验证）
- 手机从项目计划进入，能完成 里程碑 → 模块 → 明细 三层钻取查看
- 明细能在手机上 新建 / 编辑 / 变更信息 / 查看（4 个 MVP mode 全可用）
- 审批（提交 / 驳回 / 变更）在手机表单内可操作，行为与桌面一致
- 工作日自动算出的完成日期与桌面一致
- Excel 模块导入、明细导出在手机可用
- 无编辑权限的用户全程只能查看，看不到写入按钮
- 桌面 `milestone-details/**` 全部现有测试零回归
