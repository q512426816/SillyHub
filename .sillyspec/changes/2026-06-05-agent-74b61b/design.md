# Design: Agent 控制台日志回显宽度调整

## 架构决策

### 决策 1：移除页面最大宽度限制

**选择**：移除 `agent/page.tsx` 的 `max-w-6xl` 限制，让页面撑满 AppShell 主内容区

**理由**：
- Agent 控制台是监控/调试类页面，信息密度比视觉美观更重要
- 项目内其他页面（如 tasks）已使用 `max-w-[1400px]`，说明无统一宽度约束要求
- 改动最小（1行 CSS 类名），无架构风险

**Trade-off**：在超宽屏（>2560px）上，头部信息区域会显得稀疏，但可接受

### 决策 2：同时移除 mx-auto

**选择**：移除 `mx-auto`（居中对齐）

**理由**：去除 `max-w-6xl` 后，容器已填满父元素宽度，`mx-auto` 无意义

## 兼容策略

- 向后兼容：移除宽度限制不影响任何现有功能
- 响应式行为改善：在小屏设备上内容区自然收窄，大屏上获得更多空间
- `px-6` 左右内边距保留，确保内容不贴边

## 风险登记

| 风险 | 可能性 | 影响 | 缓解 |
|---|---|---|---|
| 超宽屏头部稀疏 | 低 | 低 | 可接受，后续可按需加回 max-w |
| 日志内容在超大宽度下阅读困难 | 低 | 低 | `<pre>` 元素自然折行，`word-break: break-all` 已有处理 |
