# 用户认证流程

## 目标
验证用户身份并发放短期访问凭证，确保 API 安全访问。

## 参与模块
- **backend/auth**：处理登录、token 生成、刷新验证
- **frontend**：存储 token、自动注入 Authorization header、处理 401 刷新
- **database**：存储用户凭证和会话记录

## 流程摘要

```text
┌─────────────┐
│  用户登录    │
│ (email/pwd)  │
└──────┬──────┘
       │
       ▼
┌───────────────────┐
│  验证密码          │
│  (bcrypt verify)  │
└──────┬────────────┘
       │
       ▼
┌──────────────────────┐
│  发放 Token Pair     │
│  - access (15min)    │
│  - refresh (7days)    │
└──────┬───────────────┘
       │
       ▼
┌────────────────────────┐
│  存储会话 (DB)         │
│  - hash(refresh_token) │
│  - expires_at         │
└──────┬─────────────────┘
       │
       ▼
┌──────────────────────┐
│  客户端存储          │
│  - access: memory   │
│  - refresh: secure  │
└──────────────────────┘

┌─────────────────────────────┐
│  API 请求 (access token)    │
└──────┬──────────────────────┘
       │ (15min 过期)
       ▼
┌──────────────────────┐
│  401 → 自动刷新      │
│  (用 refresh token)  │
└──────┬───────────────┘
       │
       ▼
┌─────────────────────┐
│  作废旧 refresh     │
│  发放新 token pair  │
└─────────────────────┘
```

## 失败回滚

| 失败点 | 处理 |
|--------|------|
| 密码错误 | 返回 401，审计日志记录失败尝试 |
| Token 伪造 | 返回 401，可能触发账户锁定 |
| Refresh 重放攻击 | **撤销用户所有会话**，强制重新登录 |
| 用户已停用 | 返回 403，拒绝发放 token |

## 关键术语
- **access_token**：短期 JWT（15分钟），携带用户身份和权限
- **refresh_token**：长期随机 token（7天），用于获取新 access token
- **Session**：数据库记录，存储 `hash(refresh_token)` 和过期时间
